# navigate(root, intent): the path as citation — 2026-06-08

**Status: opening design, for review.** Unlike currency, this has not yet been
simulated on a real corpus or adversarially reviewed. It states the shape and
the load-bearing boundary so codex can push back and we can put it through the
same *simulate-on-murail → review → lock* loop currency went through.

## What navigate is

The next surface of the feature arc. Where `context`/`search` return ranked
*hits* and `currency` says which are live, **navigate returns a reasoned *path*
through the typed-edge graph — and the path itself is the citation.** It answers
*"from `root`, how do I reach `intent`, and what's the trail?"* — the memex
move: associative trails are the unit of orientation, not taxonomic lookup.

It is the single artifact that most shortens a cold agent's daily orient: instead
of rebuilding the project's theory hit-by-hit (Naur), it gets a current,
provenance-linked trail in one call.

## The load-bearing boundary (bitter lesson)

This is the decision the whole design turns on:

> **navigate is a *thin projection*: anneal supplies the *current,
> provenance-linked path built from author-declared edges* (the structural
> guarantee); the *model* reasons and synthesises over it.**

The durable, model-can't-fake part is the **trail itself** — a verifiable
sequence of author-declared edges between current nodes, returned *as the
citation*. The moment the *path* is LLM-reasoned, it becomes the part a stronger
model eats (§13 substrate/model split). So navigate must never *infer* edges; it
*walks* the ones authors declared and *returns them for the model to read*.

This also keeps it on the right side of the trust invariant: the path is a GATE
(every edge author-declared, every node currency-checked) — not a synthesis
presented with more authority than its oracle earns.

## What it composes from (all existing facts)

- **Edges** — author-declared typed relations (`Cites`, `Refines`, `DependsOn`,
  `Supersedes`, parent). The graph to traverse.
- **Currency** — the path runs over **current** nodes; superseded nodes are off
  the *live* trail (but reachable for "what did this supersede" — history is kept).
- **Relevance** — `intent` scores which path, among many, to return.
- **Lifecycle** — prefer `operative` waypoints (don't route a trail through drafts).

So navigate is largely a *composition* of dimensions anneal already has, over the
edge graph — inference-free, like currency's first cut.

## First-cut hypothesis (to simulate + review)

`navigate(root, intent)` →
1. traverse author-declared typed edges outward from `root`;
2. score candidate paths by `intent` relevance × currency (live nodes) ×
   lifecycle (operative waypoints), shortest-current-relevant first;
3. return the path **as a citation** — each node with its currency disposition +
   status, each edge with its kind — so the model can read and reason over a
   trail it can verify.

Down-rank/skip superseded nodes on the live path; never *infer* a missing edge.

## Open questions (for the review + simulation)

- **`intent` scoring** — how to score "toward intent": relevance of the
  *destination*, or of the *whole path*? This is where the real difficulty may
  live (§12: the navigate verb, not the uniform store, may be doing the work).
- **Path selection** — shortest vs most-current vs most-relevant; ties; cycles
  (the corpus graph has them — `DependsOn`, mutual `Cites`).
- **Multi-trail** — the memex point is *many* trails reach the same node; should
  navigate return one path or a small fan of them?
- **Clustering** — cross-cluster trails may want community structure (the same
  topic-clustering substrate the *unmarked-currency* frontier needs). If so,
  navigate and currency-unmarked share that substrate — build it once, or defer.
- **Cold-start shape** — is `navigate` a new verb, or does it fold into
  `context` as a `--path` mode? (Surface-economy: prefer fewer verbs.)

## Acceptance shape (to make concrete after simulation)

On a real corpus (anneal's own `.design`, which now has the currency lineages):
`navigate(<a root>, <an intent>)` returns a coherent path of **current** nodes
via author-declared edges; superseded nodes are absent from the live path but
reachable by direct/chain query; every edge is verifiable (author-declared).

## Next step

Put this through currency's loop: **simulate `navigate` on a real corpus**
(does edge-traversal + intent-scoring + currency actually produce a useful trail,
and where does it break?), then codex adversarial review, then lock the first
cut. Do not implement before the simulation — that loop is what made currency
sound on the first try.
