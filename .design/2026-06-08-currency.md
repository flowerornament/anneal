# Currency: deriving the corpus's live view — 2026-06-08

The first surface of anneal's feature arc. This document defines **currency**
precisely, validates the design against a real churning corpus (murail), and
scopes a tight, inference-free first cut (`anneal-z4x3`) that fixes a verified
consumer failure using only facts anneal already has.

## 1. What currency is

**Currency is the property of a claim being the corpus's *operative word on its
subject as of now* — that no later claim has displaced it.** It answers one
question: *has this been superseded?* The English word is the tell: a *currency*
is what is **currently accepted in circulation** — legal tender. An old coin
undisplaced is still current; a crisp new note that's been demonetised is not.

> **Currency = is this claim still legal tender in the corpus?**

Five properties fix what it is, and each is a wall against a confusion:

1. **Derived, not declared.** Computed (`current(F) :- minted(F), not superseded(F)`),
   never a field. A hand-typed `status:` is a *claim about* currency that rots.
2. **Relational, not local.** A claim's currency lives in *other* claims about
   the same subject; you cannot read it off the node.
3. **Non-monotone — the essence.** Currency is the *only* retrieval dimension a
   *new* fact can *revoke* (assert a supersession → a current claim goes stale).
   This is why it is the one hard locus; relevance/recency/centrality are all
   monotone and easy.
4. **Temporal, but not recency.** It is about *displacement*, not edit-date. A
   months-old undisplaced spec is current; a doc a sweep touched yesterday is
   stale. `git-mtime` measures the wrong time entirely.
5. **Epistemic, not veridical.** Currency ≠ correctness. A current claim can be
   *wrong* (merely undisplaced); a superseded one may have been right. Currency
   tracks what the corpus *presently asserts*, not what is true.

**What it is for.** Currency is the projection that turns an append-only
**archive** into a live **belief-set**: keep everything (auditable,
re-derivable) yet read only what is operative now. It is the lens that separates
*the record* from *the current belief* — what lets a corpus grow without end and
stay consultable (Bush's selection bottleneck, dissolved). In the convergent-
substrate frame: currency is **the** non-monotone projection over the Fact log.

It is not its neighbours:

| dimension | question | a new fact can revoke it? |
|---|---|---|
| relevance | matches my query? | no |
| recency | when touched? | no |
| authority/status | what does it say about itself? | no |
| importance/hub | how connected? | no |
| **currency** | **still the operative word on its subject?** | **yes** |

## 2. The rule

**Currency is displacement, and nothing else.** An earlier draft folded a second
condition ("minted") into currency — that was a conflation. *Draft vs adopted* is
**lifecycle** (a separate axis, read off `status`), not currency. Keep them
separate; **orientation composes them.**

**Currency (edge-derived, non-monotone).** Live `Supersedes` file edges run
`{from: old, to: new}` (verified: `cell-graph→05-31`, `v16→v17`, `05-30→05-31`):

```
currency_superseded(h)   := *edge{from: h, to: _, kind: "Supersedes"}.   % h is OLD → displaced
currency_successor(h)    := *edge{from: _, to: h, kind: "Supersedes"}.   % h is NEW → displaced something
currency_current(h)      := not currency_superseded(h).                  % still legal tender (pure)
currency_current_head(h) := currency_successor(h), not currency_superseded(h).
```

**Lifecycle (self-declared, via `status` — a SEPARATE axis):**

```
operative(h) := status(h) in {active, authoritative, current, stable, living}.
% pre-operative: draft/proposed ;  retired: superseded/deprecated/abandoned
% NB: NOT the active() primitive — active() means non-terminal (admits drafts,
% and excludes authoritative: active("v17") = 0 rows).
```

**Orientation composes the two** — "the live operative word" lives *here*, not
inside currency: boost `operative(h) ∧ currency_current_head(h)`; down-rank
`currency_superseded(h)`; all within the relevance set. A draft that happens to
be a successor (`current_head` but not `operative`) does **not** get the boost.

**Why both, not one:** `status` is self-declared and lags; `Supersedes` is
relational and authoritative. A doc can read `status: active` (operative) yet be
superseded by edge (not current) — currency catches what the status field missed.

**Direction caveat (verified):** file/doc handles use old→new, but some
*synthetic/version* edges use the reverse short-id convention (`v17→v16`). Scope
the first cut to file/doc handles + old→new file edges; be defensive about the
reverse short-id edges (relates to `zbq8`). A **sibling pre-existing bug**:
`orientation_replaced(h)` keys on incoming `*edge{to: h}`, which under old→new
marks the *new* head as replaced — likely why `recent_frontier` returns 0 for
`05-31`. Fix it in the same cut.

**Disposition-typed** (per the witnesses framework — a surface earns only the
authority its oracle supports):

| disposition | meaning | oracle | action |
|---|---|---|---|
| **current** | undisplaced (legal tender) | clean | rank normally |
| **current_head** | undisplaced + displaced a prior | clean | **boost when also `operative`** |
| **superseded** | marked terminal (healthy decay) | clean (GATE) | **down-rank** |
| **suspect** | a newer authoritative sibling on the same topic, but *unmarked* | no clean oracle | **REPORT hint only — never an asserted edge** |
| **unknown** | degenerate input (no history/siblings) | none | signal, don't fake (the xy45 trust invariant) |

**Currency refines relevance; it does not replace it.** Currency is
currency-*of-a-subject*, so it acts *within* the relevant set, not as a
standalone ranker — global `¬superseded` would surface `LABELS.md` for a perf
query. The composition is: relevance selects the set; currency re-weights it
(boost `current_head`, down-rank `superseded`) and annotates each hit.

## 3. Validation — simulated on murail (real data, 2026-06-07)

murail's `.design` has real supersession lineages *and* a consumer-verified
orientation failure (perf-arc lead, bd `murail-ze8n`): the current perf framing
`2026-05-31-program-space` "falls between" `ranked_anchor` (old-authoritative)
and `recent_frontier` (newest drafts) and is missed.

Reproduced and re-ranked by hand over existing `Supersedes` edges + `active`:

| candidate | superseded? | successor-head? | minted? | verdict |
|---|---|---|---|---|
| `2026-05-31-program-space` | no | **yes** (displaced cell-graph + 05-30) | active | **current_head → boost** |
| `formal-model/v17` (cost-model §) | no | yes | authoritative | **current_head → boost** |
| `2026-05-30-unified-perf` | **yes** | yes | — | **superseded → demote** |
| `2026-03-30-cell-graph-optimization` | **yes** | no | — | **superseded → demote** |
| `LABELS.md` / `DESIGN-GOALS.md` | no | no | active | current, not a head → below |

Re-rank result: `2026-05-31` + `v17` float to the top; the two superseded specs
drop; meta sits below. **The design produces the right orientation on real data**
— and surfaces the current *set* (perf spec + formal cost model), not a forced
single winner.

**The subtlety real data caught:** `2026-05-30-unified` is *both* a successor-head
*and* superseded — it displaced an older spec, then 05-31 displaced it. So the
rule must be `successor ∧ ¬superseded`, not `successor` — `¬superseded` is
load-bearing, demoting intermediate chain-heads. Asserted from data, not assumed.

**The precise locus:** keyword search with the right title words already finds
05-31. The failure is purely in the **orientation verbs** — `ranked_anchor`
ranks by `authoritative_status`, leaving the current `05-31` at rank ~25 (score
~18) far below the meta-docs and authoritative anchors; `recent_frontier`
excludes it entirely (the `orientation_replaced` direction bug above). Neither
has a currency term. That is the fix site. (Earlier prose claimed `ranked_anchor`
surfaces the *superseded* cell-graph spec — corrected: cell-graph does **not**
appear in `ranked_anchor`; the failure is that 05-31 is missed/too-low.)

## 4. First cut (`anneal-z4x3`) — inference-free, existing facts only

1. **Currency term in the orientation ranking.** Add `current_head` /
   `superseded` to `ranked_anchor` (new `why: "current_head"`) and fold a
   currency re-weight into `context` default ranking: boost `current_head`,
   down-rank `superseded`, within the relevance set. Lives in
   `prelude/orientation.dl` + `ranking.rs`, plus the `orientation_replaced`
   direction fix. **Calibration** (from real murail scores): `current_head` ≈
   **90–120** — above curated/inbound (~65/50), below `authoritative` (~250) so
   it lifts `05-31` into view without outranking true authoritative anchors.
   The signals — `Supersedes` edges + settled status — are **already in the
   graph** (no extraction, clustering, or LLM inference) — **but** `ranking.rs`'s
   `SearchIndex` currently keeps only incoming edge *counts* and discards edge
   *kind/source*, so it must be extended to retain `Supersedes` from/to from the
   existing `EdgeFact`s for the `context`/`search` side. Still inference-free; a
   real impl requirement.
2. **Per-hit disposition legibility.** Every `context`/`search` hit shows its
   currency disposition (`current` / `current_head` / `superseded`) + status +
   age. **Age source for this slice = `*handle.date` / `freshness()`** (authored
   date, not git-mtime) — snapshot rows currently carry only `status`, so
   *content-age from snapshot history* is a future extraction feature, not z4x3.
   Annotate at the CLI/context result layer (join handle + currency facts); do
   **not** change the primitive `search(...)` signature.

**Hard constraint:** rendering a live view must **not** break access to
superseded material — "what did this supersede / what changed" stays a
first-class query. We surface a belief-set *over* a kept archive; we never delete.

### Acceptance gate (exact deterministic probes, not prose)
On murail `.design`:
- **currency probes:** `v17` `current_head=true`; `05-31` `current_head=true`;
  `05-30` `superseded=true ∧ current_head=false`; `cell-graph` `superseded=true`.
- **`ranked_anchor` ordered by rank:** `05-31` appears within a fixed top-N
  threshold; `v17` stays in the top handful.
- **`context` perf query:** `05-31` ranks above `05-30`; dispositions visible.
- **non-currency surfaces byte-identical** where currency doesn't apply.
- **avoid `--explain` over synthetic `Supersedes` ordinals** (`zbq8`
  nondeterminism) — gate on the deterministic probes above.

## 5. Deferred frontier (off the critical path — proven by the sim)

- **mint-rot** — `minted` leans on `status:`, which rots; a draft whose status
  prematurely reads `active` would false-surface. This is the *genuine* hard
  edge, more than unmarked detection. Make "minted" more derived (graduation as
  an event, not a field) — later.
- **unmarked supersession** (the `suspect` disposition) — needs topic/community
  clustering to find "a newer sibling on the same topic." The marked edges carry
  the whole win on the verified case; defer the clustering substrate until a
  surface needs it. Clustering, when built, is also what `navigate` consumes.
- **hub-ness / centrality** — a *separate* signal (the consumer's second ask).
  Redundant for the perf case (05-31 is a hub *and* a head; currency surfaces
  it). Add as its own ranking term later if a current-but-unsuperseded hub needs
  lifting.

## 6. Open questions

- **Granularity** — handle/doc-level currency is adequate today (both real
  lineages supersede whole-doc-to-whole-doc; no partial supersession in the live
  corpus). Revisit claim-level only if partial supersession appears.
- **Which relations bear on currency** — `Supersedes` carries the win.
  `refines`/`extends` leave both current; **`contradicts`** is the interesting
  one (two live claims in conflict → *both* `suspect`, though neither is
  superseded) — currency may not be purely about displacement. Future.
- **Altitudes (the fractal test)** — currency of an *edge* (citation to a
  now-superseded target), a *trail*, a *goal*. The convergent-substrate doc bets
  one projection serves all; doc-currency is the only clean case proven so far.
- **Bitemporal** — currency is present-tense by default; anneal's snapshot
  history already permits `current_as_of(T)` as a time-cut query. Powerful, later.

## 7. The frame this sits in

Currency is anneal's first move into the **substrate/model split**: it is a
*left-column* guarantee — *what is current, derived not self-reported* — exactly
what a stronger model cannot certify for itself and must consume from a
structural source. `navigate` (a thin projection returning a current,
provenance-linked path the model reasons over) and the TMS far-end (retraction
propagation) are later surfaces on the same engine — the planned Datalog runtime
this codebase already is. Currency is where the durable value concentrates
because it is the one non-monotone projection; everything else is the easy
monotone fragment riding on top.
