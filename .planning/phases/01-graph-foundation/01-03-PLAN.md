**1. Namespace inference** (HANDLE-05, spec KB-D4):
```rust
pub fn infer_namespaces(
    candidates: &[LabelCandidate],
    config: &AnnealConfig,
) -> HashSet<String>
```

Algorithm:
1. Start with `config.handles.confirmed` as already-confirmed namespaces
2. Build `rejected: HashSet<String>` from `config.handles.rejected`
3. Group candidates by prefix: `HashMap<String, Vec<(u32, Utf8PathBuf)>>`
4. For each prefix not in confirmed or rejected:
   - Compute N = count of distinct sequential numbers (use `BTreeSet<u32>`)
   - Compute M = count of distinct file paths (use `HashSet<&Utf8PathBuf>`)
   - If N >= 3 AND M >= 2 -> add to confirmed (HANDLE-05)
   - Additional heuristic: reject prefixes where the minimum number is large (>100) AND there are no sequential runs (e.g., SHA-256 has only number 256). A "sequential run" means at least 3 consecutive numbers exist in the set. This catches SHA-256, AVX-512, CRC-32 etc. that have only 1-2 distinct numbers.
5. Return the confirmed set

**2. Label resolution** (HANDLE-03, HANDLE-06):
```rust
pub fn resolve_labels(
    graph: &mut DiGraph,
    candidates: &[LabelCandidate],
    namespaces: &HashSet<String>,
) -> ResolveResult
```

- For each candidate whose prefix is in `namespaces`:
  - Create or look up Label handle: `HandleKind::Label { prefix, number }`
  - The identity string is `"{prefix}-{number}"` (e.g., "OQ-64")
  - Add node to graph if not already present (use a `HashMap<String, NodeId>` to track)
  - Create edge from the file node (candidate's source) to the label node with the candidate's `edge_kind`
- Skip candidates whose prefix is NOT in confirmed namespaces (HANDLE-06: no diagnostics for unconfirmed)
- Return statistics: how many labels resolved, how many skipped


**3. Version handle resolution** (HANDLE-04, spec KB-D2, KB-D3):
```rust
pub fn resolve_versions(
    graph: &mut DiGraph,
    node_index: &HashMap<String, NodeId>,
) -> usize
```

Version handles resolve by matching versioned artifact naming conventions. Scan existing File handles in the graph for files matching the pattern `*-v{N}.md` (e.g., `formal-model-v3.md`, `proof-v17.md`). For each match:
1. Extract the base name (without `-vN.md`) and the version number N
2. Look up the file's NodeId from the graph
3. Create a Version handle node: `HandleKind::Version { artifact: file_node, version: N }` with id `"{base}-v{N}"`
4. Add the version node to the graph
5. Add a `Supersedes` edge from the version node to the previous version if it exists (e.g., v3 supersedes v2, if v2 is in the graph)

Use a regex like `r"^(.+)-v(\d+)\.md$"` applied to file names (not full paths). Group versioned files by base name, sort by version number, and create the supersession chain.

Return count of version handles created.

**4. Pending edge resolution** (HANDLE-01, HANDLE-02):
```rust
pub fn resolve_pending_edges(
    graph: &mut DiGraph,
    pending: &[PendingEdge],
    node_index: &HashMap<String, NodeId>,
) -> usize
```

- For each pending edge, look up `target_identity` in the `node_index`
- If found: create edge via `graph.add_edge(source, target_id, kind)`
- If not found: skip (Phase 2's CHECK-01 will report broken references)
- Return count of edges resolved

The `node_index` is a `HashMap<String, NodeId>` mapping handle identity strings to NodeIds. Build this from the graph's nodes:
- File handles: identity = relative path string
- Section handles: identity = "heading text" or "{file}#heading"
- Label handles: identity = "PREFIX-NUMBER"
- Version handles: identity = "{base}-vN"

**2. Main pipeline** (the core of Phase 1):
```rust
fn main() -> anyhow::Result<()> {
    let cli = Cli::parse();

    // 1. Determine root
    let cwd = Utf8PathBuf::try_from(std::env::current_dir()?)?;
    let root = if let Some(ref r) = cli.root {
        Utf8PathBuf::from(r)
    } else {
        parse::infer_root(&cwd)
    };

    // 2. Load config (zero-config is valid)
    let config = config::load_config(&root)?;

    // 3. Build raw graph (scan files, parse frontmatter, scan content)
    let (mut graph, candidates, pending) = parse::build_graph(&root, &config)?;

    // 4. Resolve handles (namespace inference, label nodes, version nodes, pending edges)
    let stats = resolve::resolve_all(&mut graph, candidates, pending, &config, &root)?;

    // 5. Infer lattice from observed statuses
    let observed_statuses: HashSet<String> = graph.nodes()
        .filter_map(|(_, h)| h.status.clone())
        .collect();
    let lattice = lattice::infer_lattice(&observed_statuses, &config);

    // 6. Print results
    if cli.json {
        // JSON output (GRAPH-06 compatible -- just stats, no graph persistence)
        let output = GraphSummary { ... };
        println!("{}", serde_json::to_string_pretty(&output)?);
    } else {
        // Human output
        print_summary(&graph, &stats, &lattice);
    }

    Ok(())
}
```
