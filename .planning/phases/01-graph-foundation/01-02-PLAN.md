**3. RegexSet with LazyLock** (5 patterns per spec KB-D6 section 5.1):
```rust
static PATTERN_SET: LazyLock<RegexSet> = LazyLock::new(|| {
    RegexSet::new([
        r"^#{1,6}\s",                    // 0: section headings
        r"[A-Z][A-Z_]*-\d+",            // 1: label references
        r"§\d+(?:\.\d+)*",              // 2: section cross-references
        r"[a-z0-9_/-]+\.md\b",          // 3: file path references
        r"\bv\d+\b",                     // 4: version references
    ]).expect("regex patterns must compile")
});
```

Plus individual capture regexes in LazyLock:
- `LABEL_RE`: `r"([A-Z][A-Z_]*)-(\d+)"` to extract prefix and number
- `SECTION_REF_RE`: `r"§(\d+(?:\.\d+)*)"` to extract section number
- `FILE_PATH_RE`: `r"([a-z0-9_/-]+\.md)\b"` to extract file path
- `VERSION_RE`: `r"\bv(\d+)\b"` to extract version number
- `HEADING_RE`: `r"^(#{1,6})\s+(.+)"` to extract heading level and text

Scanning logic:
- Split frontmatter first, scan body only
- Track `in_code_block: bool` -- toggle on lines starting with ``` (per Pitfall 3)
- For each body line, check against `PATTERN_SET`
- If inside code block, skip heading detection (pattern 0) but still detect labels/refs
- For matched patterns, use individual regex to extract captures
- Headings (pattern 0): create Section handle via `graph.add_node()` with `HandleKind::Section { parent: file_node, heading }`
- Labels (pattern 1): collect as `LabelCandidate` (NOT added to graph yet -- namespace inference happens in resolve.rs)
- Section refs (pattern 2): collect as strings for later edge creation
- File paths (pattern 3): collect as strings for later edge creation
- Version refs (pattern 4): collect as u32 for later resolution in resolve.rs (HANDLE-04)

**5. Edge kind inference from context** (per D-01, locked decision):

Frontmatter fields -> edge kind (applied during frontmatter parsing):
- `superseded-by:` value -> create edge with `EdgeKind::Supersedes` from the superseding file to this file
- `depends-on:` values -> create edges with `EdgeKind::DependsOn`
- `discharges:` values -> create edges with `EdgeKind::Discharges`
- `verifies:` values -> create edges with `EdgeKind::Verifies`

Body text keyword proximity (same-line rule per research recommendation):
```rust
static DEPENDS_ON_KEYWORDS: &[&str] = &["incorporates", "builds on", "extends", "based on"];
static CITES_KEYWORDS: &[&str] = &["see also", "cf.", "related"];
```
- When a label/file/section reference is found on a line, check if the same line contains any keyword
- If a DependsOn keyword is found -> `EdgeKind::DependsOn`
- If a Cites keyword is found -> `EdgeKind::Cites`
- Default (no keyword) -> `EdgeKind::Cites`

pub enum ConvergenceState {
    Exists,     // two-element lattice: we know it exists
    Missing,    // two-element lattice: referenced but not found
    Active(String),    // confidence lattice: active status value
    Terminal(String),  // confidence lattice: terminal status value
}

**2. Lattice struct** holding the computed lattice for a corpus:
```rust
#[derive(Debug, Serialize)]
pub struct Lattice {
    pub observed_statuses: HashSet<String>,
    pub active: HashSet<String>,
    pub terminal: HashSet<String>,
    pub ordering: Vec<String>,   // empty if flat set (no pipeline ordering)
    pub kind: LatticeKind,
}

#[derive(Debug, PartialEq, Eq, Serialize)]
pub enum LatticeKind {
    Existence,    // zero-config: {exists, missing} only (LATTICE-01)
    Confidence,   // has frontmatter status values (LATTICE-02)
}
```

**3. Lattice inference function** (LATTICE-02):
```rust
pub fn infer_lattice(
    observed_statuses: &HashSet<String>,
    config: &AnnealConfig,
) -> Lattice
```
- If `observed_statuses` is empty -> return existence lattice (`LatticeKind::Existence`, empty active/terminal)
- Otherwise -> confidence lattice:
  - Start with config overrides: `config.convergence.active` -> active set, `config.convergence.terminal` -> terminal set
  - For remaining unclassified statuses, apply directory convention heuristic (LATTICE-03):
    - This function doesn't know directories directly, so accept an additional parameter `terminal_by_directory: &HashSet<String>` -- statuses observed only in terminal directories (archive/, prior/, history/)
  - Any status not classified by config or directory convention defaults to active
  - Copy `config.convergence.ordering` as the ordering (empty if not configured)

**4. Active/terminal partition** (LATTICE-03):
```rust
pub fn classify_status(
    status: &str,
    lattice: &Lattice,
) -> ConvergenceState
```
- If status is in `lattice.active` -> `ConvergenceState::Active(status.to_string())`
- If status is in `lattice.terminal` -> `ConvergenceState::Terminal(status.to_string())`
- Default: `ConvergenceState::Active(status.to_string())` (unrecognized statuses are active)

**5. Freshness computation** (LATTICE-04, KB-D11):
```rust
#[derive(Debug, Clone, Serialize)]
pub struct Freshness {
    pub days: i64,
    pub level: FreshnessLevel,
}

#[derive(Debug, Clone, PartialEq, Eq, Serialize)]
pub enum FreshnessLevel {
    Fresh,
    Warn,
    Stale,
}

pub fn compute_freshness(
    updated: Option<chrono::NaiveDate>,
    mtime: Option<chrono::NaiveDate>,
    config: &FreshnessConfig,
) -> Freshness
```
- Use `updated` frontmatter field if present, otherwise fall back to `mtime`
- If neither available, return `Freshness { days: 0, level: Fresh }` (can't compute)
- Compute days since today: `chrono::Local::now().date_naive() - date`
- `days < config.warn` -> Fresh
- `days < config.error` -> Warn
- `days >= config.error` -> Stale
