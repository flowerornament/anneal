   **RefHint enum** (derives: Clone, Debug, PartialEq, Eq, Serialize):
   - `Label { prefix: String, number: u32 }` -- label reference like "OQ-64"
   - `FilePath` -- file path like "foo.md" or "subdir/bar.md"
   - `SectionRef` -- section cross-reference like "section:4.1"
   - `External` -- URL like "https://example.com"
   - `Implausible { reason: String }` -- rejected: absolute path, prose, wildcard, etc.

   **RefSource enum** (derives: Clone, Debug, PartialEq, Eq, Serialize):
   - `Frontmatter { field: String }` -- from a frontmatter field
   - `Body` -- from body text scanning

   **DiscoveredRef struct** (derives: Clone, Debug, Serialize):
   - `raw: String` -- original string as found
   - `hint: RefHint` -- classification
   - `source: RefSource` -- where discovered
   - `edge_kind: EdgeKind` -- what edge type
   - `inverse: bool` -- if true, actual edge is target -> source

   **FileExtraction struct** (derives: Clone, Debug, Serialize):
   - `status: Option<String>` -- frontmatter status value
   - `metadata: HandleMetadata` -- extracted metadata
   - `refs: Vec<DiscoveredRef>` -- all discovered references
   - `all_keys: Vec<String>` -- all frontmatter keys observed

   **LABEL_RE static** -- duplicate the label regex from parse.rs since extraction.rs must not import from parse.rs (avoids circular dependency). Use `LazyLock<Regex>` with pattern `r"^([A-Z][A-Z_]*)-(\d+)$"` (note: anchored with ^ and $ for exact match, unlike parse.rs's unanchored version which is for scanning body text).

   **SECTION_REF_RE static** -- `LazyLock<Regex>` with pattern `r"^section:(\d+(?:\.\d+)*)$"` for exact section ref matching.

   **`pub(crate) fn classify_frontmatter_value(value: &str) -> RefHint`** implementing the classification rules in this order (first match wins):
   1. Trim whitespace from value
   2. URL: `starts_with("http://") || starts_with("https://")` -> `RefHint::External`
   3. Absolute path: `starts_with('/') || starts_with("~/")` -> `RefHint::Implausible { reason: "absolute path".into() }`
   4. Wildcard: `contains('*') || contains('?')` -> `RefHint::Implausible { reason: "wildcard pattern".into() }`
   5. Freeform prose: `contains(' ') && !ends_with(".md") && !LABEL_RE.is_match(trimmed)` -> `RefHint::Implausible { reason: "freeform prose".into() }`
   6. Comma-separated list: `contains(", ") && len > 40` -> `RefHint::Implausible { reason: "comma-separated list".into() }`
   7. Section ref: `SECTION_REF_RE.is_match(trimmed)` -> `RefHint::SectionRef`
   8. Label: `LABEL_RE.captures(trimmed)` extracts prefix and number -> `RefHint::Label { prefix, number }`
   9. File path: `ends_with(".md")` -> `RefHint::FilePath`
   10. Default: `RefHint::FilePath` (treat as potential handle identity)

   Note on step 8: Parse the number with `caps[2].parse::<u32>()`. If parse fails (number too large), fall through to default.

   **HandleMetadata must derive Clone** -- check if it already does. If not, add `Clone` to its derive list in src/handle.rs. FileExtraction needs Clone and HandleMetadata is a field.
