    - LineIndex::from_content("line1\nline2\nline3") builds a sorted vec of newline byte offsets
    - LineIndex::offset_to_line(0) returns 1 (first line)
    - LineIndex::offset_to_line(6) returns 2 (second line of "line1\nline2\n")
    - LineIndex::from_content_with_offset("line1\nline2\n", 10) where offset=10 means body starts at file line 10; offset_to_line(0) returns 10
    - LineIndex::offset_to_line on offset beyond content returns last line number
    - Empty content: LineIndex::from_content("") with offset_to_line(0) returns 1
  </behavior>
  <action>
    Add to src/extraction.rs:

    1. SourceSpan struct:
    ```rust
    #[derive(Clone, Debug, PartialEq, Eq, Serialize)]
    pub(crate) struct SourceSpan {
        pub(crate) file: String,
        pub(crate) line: u32,
    }
    ```

    2. LineIndex struct:
    ```rust
    pub(crate) struct LineIndex {
        /// Byte offsets where each newline occurs in the content.
        newline_offsets: Vec<usize>,
        /// 1-based line number of the first byte (accounts for frontmatter).
        base_line: u32,
    }
    ```

    3. LineIndex::from_content(content: &str, frontmatter_line_count: u32) -> Self
       - Scan content for '\n' bytes, collect their byte offsets into a sorted Vec
       - Set base_line = frontmatter_line_count + 1 (the +1 accounts for the closing --- line)
       - If frontmatter_line_count == 0, base_line = 1

    4. LineIndex::offset_to_line(&self, byte_offset: usize) -> u32
       - Binary search newline_offsets for byte_offset
       - The number of newlines before byte_offset gives the 0-based line within body
       - Add base_line to get file-relative 1-based line number
       - Use partition_point (binary search) for O(log n) lookup

    5. Add `pub(crate) span: Option<SourceSpan>` field to DiscoveredRef (after `inverse`).
       Default to None. This is additive-only -- existing construction sites pass None.
       Update the DiscoveredRef construction in parse.rs build_graph (the frontmatter loop) to pass `span: None`.

    6. Write tests in extraction.rs::tests covering all behavior cases above.
