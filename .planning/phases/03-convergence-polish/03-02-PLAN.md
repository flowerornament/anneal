  <action>
    Add five suggestion functions to `src/checks.rs`, each returning `Vec<Diagnostic>` with `Severity::Suggestion`:

    **S001 - Orphaned handles** (SUGGEST-01, per D-17):
    ```rust
    fn suggest_orphaned(graph: &DiGraph) -> Vec<Diagnostic>
    ```
    Iterate `graph.nodes()`. For each non-File handle, check if `graph.incoming(node_id).is_empty()`. If yes, emit Diagnostic with severity=Suggestion, code="S001", message="orphaned handle: {id} has no incoming edges". Skip File handles entirely (they are roots per D-17).

    **S002 - Candidate namespaces** (SUGGEST-02):
    ```rust
    fn suggest_candidate_namespaces(graph: &DiGraph, config: &AnnealConfig) -> Vec<Diagnostic>
    ```
    Collect all Label handle prefixes. Group by prefix. For prefixes NOT in `config.handles.confirmed` and NOT in `config.handles.rejected`, if count >= 3 (cardinality threshold matching namespace inference), emit S002: "candidate namespace: {prefix} ({count} labels found, not in confirmed namespaces)". One diagnostic per candidate prefix (not per label).

    **S003 - Pipeline stalls** (SUGGEST-03, per KB-E4):
    ```rust
    fn suggest_pipeline_stalls(graph: &DiGraph, lattice: &Lattice) -> Vec<Diagnostic>
    ```
    If `lattice.ordering.is_empty()`, return empty. Group handles by their ordering level (using `lattice::state_level`). For each level except the last: count handles at that level. Count how many have at least one outgoing DependsOn edge targeting a handle at the next level (use `graph.edges_by_kind(id, EdgeKind::DependsOn)` and check target's status level). If the level has >= 3 handles and zero have DependsOn to next level, emit S003: "pipeline stall: {count} handles at status '{level_name}' with no dependencies at next level '{next_level_name}'".

    **S004 - Abandoned namespaces** (SUGGEST-04):
    ```rust
    fn suggest_abandoned_namespaces(graph: &DiGraph, lattice: &Lattice, config: &AnnealConfig) -> Vec<Diagnostic>
    ```
    Group Label handles by prefix (for confirmed namespaces). For each namespace: if ALL members have terminal status (every handle's status is in `lattice.terminal`), emit S004: "abandoned namespace: all {count} members of {prefix} are terminal". Skip namespaces with fewer than 2 members.

    **S005 - Concern group candidates** (SUGGEST-05):
    ```rust
    fn suggest_concern_groups(graph: &DiGraph, config: &AnnealConfig) -> Vec<Diagnostic>
    ```
    Build a co-occurrence map: for each File handle, collect Label handles it references (via outgoing Cites/DependsOn edges). For each pair of label prefixes that co-occur in >= 3 files, emit S005: "concern group candidate: {prefix_a} and {prefix_b} co-occur in {count} files". Skip pairs where both are already in the same concern group from config. Limit to top 5 pairs by co-occurrence count to avoid noise.

    **Update run_checks**: Add a `run_suggestions` function:
    ```rust
    pub(crate) fn run_suggestions(graph: &DiGraph, lattice: &Lattice, config: &AnnealConfig) -> Vec<Diagnostic>
    ```
    That calls all five suggest_* functions and collects results. Also update `run_checks` to call `run_suggestions` and extend the diagnostics vector. Suggestions sort after Info due to Severity ordering.
