1. In `src/resolve.rs`, add these new types after the `Resolution` enum:

```rust
/// Result of cascade resolution for a single unresolved pending edge.
#[derive(Clone, Debug)]
pub(crate) struct CascadeResult {
    /// Index into the original pending_edges slice.
    pub(crate) edge_index: usize,
    /// Candidate handle identities found by structural transforms.
    pub(crate) candidates: Vec<String>,
    /// If exactly one unambiguous match was found, the resolved NodeId.
    /// Root-prefix strip is the only strategy that produces this (it creates a graph edge).
    pub(crate) resolved: Option<NodeId>,
}
```

2. Add three private strategy functions:

`fn try_root_prefix_strip(target: &str, node_index: &HashMap<String, NodeId>, root_prefix: &str) -> (Option<NodeId>, Vec<String>)`:
- If target starts with `root_prefix` (with or without trailing `/`), strip the prefix and look up the remainder in node_index.
- Also try stripping just the first path component if it matches root_prefix.
- If exactly one match: return `(Some(node_id), vec![stripped_identity])`.
- If multiple matches (ambiguous): return `(None, vec_of_all_matching_identities)`.
- If no match: return `(None, vec![])`.

`fn try_version_stem(target: &str, node_index: &HashMap<String, NodeId>) -> Vec<String>`:
- Use `VERSION_FILENAME_RE` to parse target as `{base}-v{N}.md`.
- If it matches, scan all node_index keys for entries whose filename matches `{base}-v{M}.md` where M != N.
- Sort matches by version number descending (latest first).
- Return the matching identity strings.

`fn try_zero_pad_normalize(target: &str, node_index: &HashMap<String, NodeId>) -> Option<(NodeId, String)>`:
- Use the `LABEL_RE` regex (import from parse.rs -- need to make it `pub(crate)` if not already, OR duplicate the pattern in resolve.rs as a dedicated cascade regex).
- Actually, `LABEL_RE` in parse.rs captures `([A-Z][A-Z_]*)-(\d+)`. For zero-pad, we need the full match. Check if the number portion has leading zeros (`starts_with('0') && len > 1`).
- If so, parse the number, rebuild as `{prefix}-{number}` (no padding), look up in node_index.
- Return `Some((node_id, normalized))` or `None`.
- For compound labels like `KB-D01`, LABEL_RE captures prefix=`KB` number=`01` which misses the compound prefix. Instead, use a dedicated regex for cascade: `^([A-Z][A-Z0-9_]*(?:-[A-Z][A-Z0-9_]*)*)-0*(\d+)$` that captures the full compound prefix and the zero-padded number. Name it `COMPOUND_LABEL_RE`.

3. Add the main cascade function:

```rust
///
/// Called after `resolve_all()`. For each unresolved edge, tries strategies
/// in order: root-prefix strip, version stem, zero-pad normalize.

/// All other matches produce candidate lists for diagnostic enrichment.

    graph: &mut DiGraph,
    pending: &[PendingEdge],
    node_index: &HashMap<String, NodeId>,
    root_prefix: &str,
) -> Vec<CascadeResult> {
```

- Iterate over `pending` with index.
- For each edge, check if it's already resolved (skip if `node_index.contains_key(&edge.target_identity)`).
- Skip section refs (`edge.target_identity.starts_with("section:")`).
- Apply strategies in order, accumulating candidates:
  1. `try_root_prefix_strip` -- if resolved (unambiguous), add edge to graph, record in result
  2. `try_version_stem` -- add any results to candidates
  3. `try_zero_pad_normalize` -- if match found, add to candidates (zero-pad does NOT create edges since the label identity is canonical, only suggests the canonical form)
- Collect into `Vec<CascadeResult>` (only entries with non-empty candidates or a resolved match).

4. Make the `root_prefix` derivation clear: the caller (main.rs) derives it from `config.root` -- if root is `.design/`, the prefix is `.design/`. If root is empty or `.`, no prefix stripping applies.
